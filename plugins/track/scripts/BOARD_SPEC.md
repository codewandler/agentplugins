# Board generation spec

`gen_board.py` rewrites the generated region of `docs/stories/README.md` from story frontmatter.
This document specifies the algorithm precisely so an **agent can reproduce byte-identical output by
hand** when `python3` is unavailable. Follow it exactly.

## Inputs

- `docs-dir` (default `docs`). Stories live in `<docs-dir>/stories/`. The board is
  `<docs-dir>/stories/README.md`.

## 1. Collect stories

List `<docs-dir>/stories/*.md` in **case-sensitive ascending filename order**. Skip `README.md` and
`_TEMPLATE.md`. For each remaining file, read the leading frontmatter — the block between the first
line (`---`) and the next `---`. Parse it as `key: value` per line:

- Split on the **first** `:` only. Trim whitespace from key and value.
- If the value is wrapped in matching single or double quotes, strip them.
- Fields used: `id`, `title`, `pillar`, `status`, `priority`, `epic`. (`design` is ignored here.)

**Normalize the status** before matching: lowercase it and replace runs of whitespace/underscores
with a single hyphen (so `In Progress`, `in progress`, `in_progress` all become `in-progress`).
**Parse the priority** leniently: use the first integer found in the value (`P1` → 1, `3` → 3); if
there's no integer, treat the priority as absent.

Skip a file (with a warning) if it has no frontmatter, is missing any of `id`/`title`/`status`, or
has a normalized `status` not in `backlog | ready | in-progress | blocked | done`. Record a warning
for any duplicate `id`, for a `ready` story with no `priority`, and for a `priority` with no integer.

## 2. Bucket by status

Bucket every story by its normalized status: **Now** (`in-progress`), **Next** (`ready`), **Blocked**
(`blocked`), **Backlog** (`backlog`), **Done** (`done`).

## 3. Sort

- **Natural id order** = sort by `(uppercased-prefix, number, raw-id)` where prefix/number come from
  the leading `^([A-Za-z]+)[-_ ]?(\d+)` of the id (e.g. `D-2` before `D-10`); ids that don't match
  sort last under prefix `~`.
- **Now / Blocked / Backlog / Done**: natural id order.
- **Next**: by `(priority ascending, natural id)`. Stories with no integer priority sort after those
  with one.

## 4. Epic grouping (Next and Backlog only)

Within Next and Backlog: list stories **without** an `epic` first, in the order from step 3. Then, for
each distinct `epic` slug in **ascending slug order**, emit a blank line, a `### <Epic title>`
subheader, an optional **blurb line**, and that epic's stories (in the same sorted order). Done is a
flat list (not epic-grouped).

- **Epic title** = the H1 (`# …`) of `<docs-dir>/designs/<slug>.md` with a leading `Design:` /
  `Design —` / `Design -` stripped (case-insensitive); if that file or H1 is absent, title-case the
  slug (replace `-`/`_` with spaces).
- **Blurb** = the first non-empty, non-heading line under the design doc's `## Why` heading,
  truncated to 200 chars on a word boundary (append `…`). Render it on its own line as `_<blurb>_`.
  Omit the blurb line entirely if there's no design doc or no `## Why` content.

## 5. Render

Each story row, joining the present parts with ` · ` (U+00B7); the em dash in the link is ` — `
(U+2014):

```
- [<id> — <title>](<filename>) · <pillar> · <note>
```

Omit `<pillar>` and/or `<note>` when that frontmatter field is absent (so a story with neither renders
just `- [<id> — <title>](<filename>)`).

Build the block (note the exact headings and the blank line between sections):

```
<!-- Generated by /track:board (track plugin) from story frontmatter. Do not hand-edit this region; edit the stories and re-run /track:board. -->

## Now (in progress)
<rows, or "_None._" if empty>

## Next (ready — take the top one unless the user named a story)
<grouped rows, or "_None._" if empty>

## Blocked
<rows, or "_None._" if empty>

## Backlog
<grouped rows, or "_None._" if empty>

## Done
<done rows in natural id order, or "_None yet._" if empty>
```

If there is at least one done story, append — after a blank line following the done rows — the line
`_See [CHANGELOG.md](../../CHANGELOG.md) for the full released history._`

## 6. Splice

In `<docs-dir>/stories/README.md`, replace everything **between** the `<!-- BEGIN track:board -->`
and `<!-- END track:board -->` marker lines with: a newline, the rendered block, a newline. Keep the
marker lines and all text before/after them unchanged. If the markers are absent, do not guess — tell
the user to run `/track:init` (which seeds them). Writing is idempotent: re-running yields no change.
